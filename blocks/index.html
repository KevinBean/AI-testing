              Getting Started

              This system helps you manage technical content using several key components:

              • Blocks - Reusable pieces of technical content written in Markdown
              • References - Citations to references, papers, reports, or other documents
              • Documents - Complete documents composed of paragraphs that can reference your blocks
              • Collections - Custom groups of blocks and documents organized for specific purposes
              • Actions - AI-powered operations configured to process your content
              • Workflows - Sequences of actions that can be run together
              Workflow Example
              1. Create references in the References tab
              2. Create blocks in the Blocks tab and associate them with references
              3. Create documents that reference your blocks
              4. Set up AI-powered actions to analyze or transform your content
              5. Build workflows to automate sequences of actions
              Tip: Use the Dashboard to quickly search across all content, access common functions, and view system statistics.

              Diagrams & Math Support
              Mermaid Diagrams

              You can create diagrams using Mermaid syntax:

              ```mermaid
graph TD;
  A[Start] --> B{Is it working?};
  B -->|Yes| C[Celebrate!];
  B -->|No| D[Fix it];
```

              Example Output:

              graph TD; A[Start] --> B{Is it working?}; B -->|Yes| C[Celebrate!]; B -->|No| D[Fix it];
              Math Equations (KaTeX)

              Use $...$ for inline math and $$...$$ for display math:

              Inline Example: $E = mc^2$

              Block Example:

              $$
\int_0^\infty x^2 dx = \frac{1}{3}x^3\Big|_0^\infty

              AI Features
              Actions

              Actions are AI-powered operations you can configure to process your content:

              • Generate new content based on prompts
              • Analyze or extract information from blocks and documents
              • Modify or transform existing content
              • Synthesize information from multiple sources
              Workflows

              Workflows are sequences of actions that can be run together:

              • Chain multiple AI operations together
              • Pass results from one action to the next
              • Create complex multi-step processes
              • Save configurations for repeated use
              Note: To use AI features, you need to configure your OpenAI API key in the Settings tab.
              Advanced Search Capabilities
              Basic Search

              Simply type keywords to search across all content.

              Boolean Operators
              term1 AND term2 Find content containing both terms
              term1 OR term2 Find content containing either term (default)
              term1 NOT term2 or term1 -term2 Find content with term1 but without term2
              Exact Phrase Matching

              Use quotes for exact phrases: "exact phrase match"

              Field-Specific Search
              title:term Search only in titles
              content:term Search only in content
              tags:term Search only in tags
              reference:term Search only in references
              Combined Examples
              • title:safety AND requirements - Find content with "safety" in the title and "requirements" anywhere
              • "power system" NOT failure - Find content with the phrase "power system" but without "failure"
              • tags:important content:urgent - Find content tagged as "important" with "urgent" in the content

              All data is stored locally in your browser using IndexedDB. No data is sent to any server unless you use AI features which require API calls to OpenAI.

              Use the Export Data function in the Settings tab to save a JSON file of all your content. You can import this file later to restore your data.

              Blocks are reusable pieces of content that can be referenced by multiple documents. They are organized by references. Documents are composed of paragraphs that can include references to blocks.

              The system supports OpenAI models including GPT-3.5 Turbo, GPT-4, and GPT-4o. You'll need to provide your own OpenAI API key in the Settings tab to use these features.
